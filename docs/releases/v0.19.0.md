## 🌑 v0.19.0 — 스테이지마다 다른 밤, 적마다 다른 위협

### 🎯 핵심 변경

상용화 분석 보고서의 **Phase 1 잔여 + Phase 2 (전투 체감 강화)** 적용. 무기와 적이
"숫자 차이"가 아니라 **플레이 방식 차이**로 기억되게 만드는 데 집중했습니다.
보스전과 미니보스도 마지막 30초가 의미 있는 목표가 되도록 격노/패턴을 추가했습니다.

또한 출시 인프라가 두 단계 진전됐습니다 — **Play Games Services 활성화** (App ID
+ 리더보드 6종 실제 ID 주입, 빌드 후 PGS 로그인/점수 제출 동작), AdMob 보상형 광고
통합용 인터페이스(`AdManager`) + 결과 화면 부활/골드 2배 UI 훅 (SDK는 ID 수급 후 활성화).

---

### 🎨 스테이지마다 다른 밤

`stages.json` 의 새 `bg` 블록이 타일·자갈·반딧불·장식·횃불 색상을 스테이지마다
다르게 적용합니다. 같은 게임 같은 시스템이지만 5개 스테이지가 완전히 다른 인상.

| 스테이지 | 톤 |
|---|---|
| 메아리의 숲 | 깊은 청록 + 황록 반딧불 |
| 얼어붙은 황무지 | 차가운 청회색 + 백청 반딧불 |
| 황혼의 성소 | 자보라 + 백광 (기존 톤 기본값) |
| 화염의 협곡 | 진홍·숯빛 + 진황 반딧불 |
| 저주받은 무덤 | 검적·자색 + 핑크 반딧불 |

### 🔥 무기 두 종, 행동 패턴이 달라짐

**Fire Wisp — 랜덤 폭발 → 적 밀집도 폭발**
화면 안 적 중 K=8개 후보 위치를 점수화해서 폭발 반경 안에 가장 많은 적을 덮는
곳에 떨어집니다. 폭발이 여러 번이면 이미 맞은 적을 빼고 다음 군집 선택 →
같은 팩을 두 번 때리는 낭비가 사라집니다.

**Star Needle — 가까운 적 부채꼴 → 방향성 탄막**
- 근거리 적 무게중심 방향 우선 → 적이 흩어져 있으면 그 군집을 쓸어내는 방향
- 단일 적이면 그 방향, 적이 없으면 플레이어 **이동 방향**으로 쏨 → 피난 중에도 쏠모 있음
- needle_count 가 늘수록 부채꼴 폭이 0.32 → 1.1 라디안으로 자동 확장

### ⚠ 적 텔레그래프 강화 — "왜 죽었지" 가 사라짐

- **Dasher** — 돌진 직전 0.55초 동안 플레이어 방향으로 주황 라인이 점점 길어짐
- **Caster** — 발사 0.45초 전 마젠타 조준선이 켜지고 플레이어 이동에 실시간 반응
- **Splitter** — HP 35% 이하부터 마젠타 펄스, 사망 시 분열과 함께 외향 링 버스트
- 셋 다 텔레그래프 동안 데미지 플래시가 텔레그래프 색을 덮지 않도록 보정

### 👹 미니보스 — 새 패턴 1개

기존의 돌진형 위협 위에 **방사형 충격파**가 추가됐습니다.
- 6.5초마다 미니보스에서 마젠타 링이 천천히 확장 (1.4초 동안 360px까지)
- 안쪽 영역은 안전. 외향 링 두께(±36px) 안에 들어가 있을 때만 12 피해
- 답이 명확한 패턴 — "링이 나를 지나가기 전에 한 발 비켜라"

---

### 🩸 보스전 — 마지막 30초가 진짜 목표

HP **30% 이하** 진입 시 보스가 격노합니다.
- 오라가 진홍, 스파이크 회전 2.6배, 본체 펄스
- 접촉 피해 ×1.45, 이동 속도 ×1.35
- 3.2초마다 **8발 방사형 탄막**
- 진입 순간 외향 버스트 + 사운드 큐로 "이벤트" 화

이제 보스는 HP 바 큰 적이 아니라 마지막 30초에 다시 한 번 긴장이 올라가는 마디입니다.

### 📺 보상형 광고 — UI 자리부터 (SDK는 ID 수급 후)

[docs/COMMERCIALIZATION_ANALYSIS.md](docs/COMMERCIALIZATION_ANALYSIS.md) §5.4 권고와
Play 정책에 맞춰 **보상형 광고만** 도입합니다. 전면/배너/자동 광고는 사용하지 않습니다.

이번 버전에는 인터페이스와 UI 훅이 들어갔습니다 — SDK 통합은 ID 수급 후 다음 버전:

| 상황 | CTA | 보상 |
|---|---|---|
| 사망 후 결과 화면 | "부활하기 (광고 시청)" | HP 50% + 무적 3초 + 가까운 적 제거 → 게임 재개. **1회/run** |
| 사망/승리 후 결과 화면 | "골드 2배 (광고 시청)" | 이번 판 골드 × 2 (count-up과 다음 목표 라인 즉시 갱신). **1회/run** |

`AdManager.ENABLED = false` 상태이므로 현재 빌드에서는 두 버튼이 숨겨집니다. SDK
통합 후 자동으로 활성화됩니다. 자세한 절차는 [docs/ADMOB_SETUP.md](docs/ADMOB_SETUP.md).

### 🏆 Play Games Services 활성화

Play Console PGS 프로젝트가 출시되어 실제 App ID와 리더보드 6종 ID가 코드에 주입됐습니다
(`game_services_ids.xml`, `LeaderboardManager.LEADERBOARD_IDS`). v0.19.0 빌드부터
Android 실기기에서 PGS 로그인 → 스테이지 클리어 시간/누적 처치 점수 제출이 동작합니다.
OAuth 동의 화면은 현재 테스트 모드라 Play Console에 등록된 테스터 계정만 로그인됩니다.
