## 🌙 v0.28.0 — 스테이지 차별화 Phase 1 (팔레트 + 적 톤)

### 🎯 핵심 변경

"두 번째 스테이지인데 배경이 똑같은 것 같다"는 피드백을 받고 5개 스테이지의 hue를 명확히 분리했음. 같은 적 종류라도 스테이지마다 다른 색으로 보이게 하는 **enemy tint 시스템**도 도입. 자산 추가 없이 데이터/코드 수정만으로 끝나는 1단계.

자세한 작업 단위는 [docs/STAGE_DIFFERENTIATION_PLAN.md](../STAGE_DIFFERENTIATION_PLAN.md) 참고 — 이번 릴리즈는 그 문서의 Stage A 완료분.

### 🎨 5개 스테이지 배경 팔레트 재조정

이전까지 보이드/타일/펩블 RGB가 채널 하나에만 5~10% 차이가 있는 정도라 폰 화면에서 "전부 어두운 색"으로 보이던 문제. 각 스테이지를 하나의 hue로 통일성을 유지하면서 채도/명도 차이를 크게 키웠음.

| 스테이지 | hue | 핵심 변화 |
|---|---|---|
| Forest of Echoes | 녹 / 이끼 | 기존 톤 강화 (base) |
| Frozen Wastes | 얼음 / 푸른 | void · tile · pebble 전부 청색 계열로 통일 |
| Twilight Sanctum | 보라 / 마법 | tile 보라색 + **orange torch_glow** 콘트라스트 |
| Inferno Chasm | 불 / 주홍 | tile R채널 0.48 → 0.62, torch_glow 점화 강도 ↑ |
| Cursed Tomb | 자 / 분홍 / 죽음 | tile (R0.42 G0.26 B0.36) → (R0.52 G0.22 B0.42) |

`stages.json` schema version 7 → 8.

### 👹 스테이지별 enemy tint 시스템

`stages.json` 각 entry에 `enemy_tint: [r,g,b,a]` 신규 필드를 두고, `EnemySpawner`가 `setup()` 시점에 캐싱한 뒤 적 spawn 시 `modulate` 에 적용. 같은 슬라임이라도 스테이지마다 다른 색으로 보임.

| 스테이지 | tint | 효과 |
|---|---|---|
| Forest | `[1.00, 1.00, 1.00, 1.0]` | 변화 없음 (base) |
| Frozen Wastes | `[0.70, 0.85, 1.10, 1.0]` | 청록빛 — 얼어붙은 느낌 |
| Twilight Sanctum | `[0.85, 0.70, 1.10, 1.0]` | 보랏빛 — 마법 들린 느낌 |
| Inferno Chasm | `[1.20, 0.75, 0.65, 1.0]` | 주홍빛 — 그을린 느낌 |
| Cursed Tomb | `[1.05, 0.70, 0.85, 1.0]` | 자홍빛 — 부패한 느낌 |

`register_enemy()` 경로에서 modulate를 적용해 **Splitter가 자체 instantiate하는 splitterling 자식들도 같은 hue를 입음**. Forest처럼 `[1,1,1,1]` 인 스테이지는 modulate 호출 자체를 스킵해 분기 최소화.
