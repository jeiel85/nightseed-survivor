## 🌙 v0.32.0 — 스토리 화면 자산 리뉴얼

### 🎯 핵심 변경

v0.31.0의 "고대 장부" 1차 리디자인(절차적 양피지 + 컬러 코드)을 **사용자가 직접 만든 그림 자산 14종**으로 확장했습니다. 다크 챔버 배경, 헤더의 책 아이콘, 카드별 황금 wax seal 인장, 잠금 카드를 가로지르는 봉인 체인, 나무판 푸터 버튼까지 — 메뉴 → 스토리 화면이 단순 리스트가 아니라 "오래된 마법서의 한 장"처럼 보이도록 다듬었습니다.

전 자산은 크로마 키 그린 RGB로 작성된 것을 PIL + numpy로 알파 변환했습니다. 진짜 어두운 녹색(예: forest 인장 내부 배경)은 임계점(greenness 150~220)으로 보존됩니다.

---

### 🛠 다듬기 — StoryUI 화면

- **배경** — 절차적 별/원 `_draw()`를 제거하고 `bg_story_chamber.png`를 `STRETCH_KEEP_ASPECT_COVERED`로 깔았습니다. 35% 디밍 오버레이로 가운데 카드 가독성 확보.
- **헤더** — 책 아이콘 좌측, "스토리" 타이틀 가운데, 보이지 않는 96×96 spacer 우측의 대칭 구조. 헤더 panel은 단순 다크 plaque + 골드 underline. (Portrait 프레임 자산은 가로 헤더에서 9-slice가 부자연스러워 보류.)
- **카드 인장** — 각 스테이지의 텍스트 글리프 (♣/✦/▲ 등)를 텍스처 인장으로 교체. `stage_id` → seal png 매핑 dict 기반. 잠금 카드의 인장은 `modulate(0.45, 0.45, 0.50, 0.65)`로 desaturate.
- **잠금 카드 체인** — body 위·아래에 `chain_story_locked.png` 가로 띠. 기존 LOCKED 텍스트 + lock 아이콘은 유지.
- **푸터 나무 버튼** — "메뉴로 / 용어집" 두 버튼을 `btn_story_wood.png` 9-slice로 교체. 텍스처를 2172×724 → 543×181로 다운스케일하여 280×72 작은 버튼에서도 9-slice 마진 합이 컨테이너보다 작도록 조정했습니다 (Godot은 마진이 컨테이너보다 크면 텍스처 자체를 그리지 않음).

---

### 🛠 다듬기 — 자산 처리

- 크로마 키 알파 변환을 보수적 임계(`greenness 150 → 알파 255` / `220 → 알파 0`) + despill로 처리. 작은 인장 내부의 dark green이 누락되지 않습니다.
- 모든 자산은 `_try_load_texture()` 경로 → 자산 누락 시에도 화면이 멈추지 않고 폴백 스타일로 렌더링.

---

### 📌 알려진 사항

- `frame_story_gold.png`는 본 릴리즈에서 코드 참조 없이 자산만 보존됩니다. portrait 1161×1355 비율이라 landscape 헤더에서 9-slice 안쪽 골드 라인이 doubled 형태로 stretch되어 어색했기 때문 — 다음 버전(v0.33+) 챕터 인트로 또는 잠금 카드 outer 장식 용도로 재활용 예정.
