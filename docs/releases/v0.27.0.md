## 🌙 v0.27.0 — 캐릭터 시그니처 패시브 + 난이도 재조정 + PGS/AdMob 활성화

### 🎯 핵심 변경

각 캐릭터의 기본 무기는 그대로 두고 자동 발동되는 **고유 시그니처 패시브**를 하나씩 추가. Forest 스테이지 난이도 곡선을 초반은 완화, 후반은 압박 강화 방향으로 재조정. 그리고 헤드리스 빌드에서 누락되던 **PGS/AdMob native 통합 manifest meta-data**를 정적 주입으로 해결 — 리더보드 / 보상형 광고가 실제로 동작.

---

### ⚔️ 캐릭터 시그니처 패시브 (Phase Class-1)

각 캐릭터에 고유 자동 패시브 1종 추가. 무기 시스템과 분리된 별도 레이어로, 기존 weapon evolution / 상점 패시브와 자연스럽게 곱셈으로 합성.

| 캐릭터 | 패시브 | 조건 / 효과 |
|---|---|---|
| Vagrant | **칼날 무용** (Blade Dance) | 처치 5회마다 +5% DMG · 8초 지속 · 3중첩 |
| Spirit Sister | **영혼의 메아리** (Soul Echo) | HP <50%: magnet +60 / 풀피: CD −4% |
| Hunter | **재장전 도약** (Flee & Reload) | 적에서 멀어지는 중: CD −12% / 정지 시: DMG +15% (0.3s hysteresis) |
| Berserker | **무모한 분노** (Reckless Fury) | 피격 시 +8% DMG · 4초 · 5중첩 (피격마다 타이머 갱신) |
| Pyromancer | **재연소** (Ember Renewal) | 3발마다 +5HP (run당 max_hp 50% 상한) / HP<max: CD −3% |

- **구조** — `WeaponBase.fired` 신호 + `WeaponManager.weapon_fired` bubble + `passive_damage_mult` / `passive_cooldown_mult` 필드. 캐릭터마다 `passive_id`만 갈아끼우면 동작
- **CharacterSelect**에서 카드별 시그니처 이름·효과 노출

### 📉 난이도 곡선 재조정 — Forest

사용자 피드백: "초반이 너무 어렵고 갈수록 너무 쉬워져서 가만히 두었다가 경험치만 주으러 다니는 패턴". 다음과 같이 양 끝을 손봤음.

| 구간 | 이전 | 이후 | 효과 |
|---|---|---|---|
| Wave 0 spawn interval | 1.3s | **1.8s** | 0:00~0:35 슬라임 등장 −28% |
| Wave 1 등장 시점 | 20s | **35s** | 첫 강화 받을 시간 +15s |
| Wave 8 (3:45) count | 4 | **5** | 후반 압박 +25% |
| Wave 9 (4:15) interval | 0.65s | **0.50s** | 최종 spawn rate +30% |
| Wave 10 (보스 동반) count | 4 | **5** | 보스 + minion 압박 강화 |
| 전체 `max_enemies` 캡 | 200 | **280** | 후반 spawn rate를 캡으로 흡수하던 평탄화 제거 |

다른 스테이지는 그대로 — Forest가 entry stage라 첫 인상이 가장 중요. 나머지는 playtest 결과 따라 같은 패턴 적용 예정.

### 🔌 PGS / AdMob native 통합 fix

v0.26.0의 "리더보드 미동작 / 보상형 광고 미재생" Known Issue 해결. 원인 진단을 정정:

- **오진**: PGS / AdMob .aar 누락 → GUI 에디터 빌드만 안정 해결
- **실제**: .aar 클래스는 dex에 들어있었으나, `_get_android_manifest_application_element_contents()` 훅이 헤드리스 export에서 발동되지 않아 **AndroidManifest의 meta-data가 통째로 누락**됨 (`com.google.android.gms.games.APP_ID`, `com.google.android.gms.ads.APPLICATION_ID`, 7개 Godot plugin singleton 등록)

수정: `godot/android/build/AndroidManifest.xml`에 PGS+AdMob APP_ID meta-data 2개 정적 추가. `build.gradle`에 plugin .aar 3개 + Maven deps (gson / play-services-games-v2 / play-services-ads) 정적 선언 (export plugin 미발동 시 안전망).

검증: 로컬 headless APK build → aapt2 dump xmltree로 meta-data + dexdump로 클래스 모두 확인.

### 🐛 기타 버그 수정

- **무기 중복 카드** — LevelUp에 같은 무기 카드 2개가 동시 노출되던 버그. 루트 원인은 `WeaponManager.add_weapon`이 `weapons.append` 후에 `add_child`를 호출해 weapon_name이 디폴트 "Weapon"인 윈도우에서 `has_weapon` 체크가 실패하던 race. 3중 방어 (add_weapon 순서 교체 + `_add_weapon` has_weapon guard + LevelUpUI `seen_up` dedup)
- **결과 패널 영문 버튼 텍스트 잘림** — "Play Agai", "Main Men" 으로 끝글자 잘리던 문제. 폰트 사이즈 축소 (32→26 / 28→24 / 24→20) + `size_flags_horizontal=3 + clip_text=true` 일괄 명시 (MainMenu 패턴과 일치)
