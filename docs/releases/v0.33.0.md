## 🌙 v0.33.0 — 스토리 상세 장부와 한국어 문장 윤문

### 🎯 핵심 변경

v0.32.0에서 리뉴얼한 StoryUI에 실제로 읽을 수 있는 상세 장부 본문을 연결했습니다. 이제 인게임에서는 짧은 진행 자막만 보여 주고, 메인 메뉴의 스토리 화면에서는 스테이지별 사건을 더 긴 연대기 형태로 읽을 수 있습니다.

---

### 🆕 추가 — Story Chronicle 상세 장부

- `godot/data/story_chapters.json` 신규 추가.
- 5개 스테이지에 한국어/영어 상세 본문 추가.
  - 스테이지 해금 시 공개되는 본문
  - 스테이지 클리어 후 공개되는 추가 단서
  - Cursed Tomb 클리어 후 공개되는 캠페인 후일담
- 공개 조건:
  - `stage_unlocked`: 스테이지 해금 시 표시
  - `stage_cleared`: 해당 스테이지 클리어 후 표시
  - `campaign_cleared`: 마지막 스테이지 클리어 후 표시

---

### 🛠 변경 — StoryUI 읽기 구조

- StoryUI 카드 구성을 `요약 → 장부 → 전투 기록` 순서로 변경.
- 기존 인게임 자막은 하단 `전투 기록` 섹션으로 축소 표시.
- 아직 클리어하지 않은 상세 섹션은 제목과 잠금 안내만 보여 주고, 본문 스포일러는 숨김.
- `story_dialogues.json`는 인게임 `StoryBanner`용 짧은 자막으로 유지.

---

### ✍️ 다듬기 — 한국어 문장 윤문

- `잊힌 맹세/약속`처럼 압축되어 보이는 표현을 문맥에 맞게 풀어 씀.
- 너무 짧게 끊겨 번역투처럼 보이던 문장을 더 자연스러운 한국어 서사 흐름으로 연결.
- Frozen Wastes 인게임 자막과 반복 힌트도 같은 톤으로 보정.

---

### ✅ 검증

- Python JSON parse: `story_chapters.json`, `story_dialogues.json`, `story_terms.json` 통과.
- `git diff --check` 통과.
- Godot headless 기본 실행 및 StoryUI 단독 로드에서 스크립트 에러 출력 없음.
  - 단, 기존과 동일하게 종료 시 ObjectDB leak 경고로 exit code 1 반환.
- StoryUI 720×1280 뷰포트 캡처 확인.
  - 헤더, 카드 폭, 긴 한국어 문장 줄바꿈, 스크롤 영역, 하단 버튼 정상.
- GitHub Actions main push 빌드 성공.
