### 📱 Android Back 키 + 일시정지 메뉴

`project.godot` 에 `application/config/quit_on_go_back=false` 를 추가해 Godot의 기본 "Back = 앱 종료" 동작을 끄고, 각 화면의 루트 스크립트가 `NOTIFICATION_WM_GO_BACK_REQUEST` 를 받아 직접 처리:

| 화면 | Back 동작 |
|---|---|
| 전투 중 (GameRoot) | 일시정지 메뉴 열기 (계속하기 / 메인 메뉴로) |
| 메인 메뉴 | "Nightseed Survivor 를 종료할까요?" 확인 다이얼로그 |
| 캐릭터 / 스테이지 / 상점 / 코덱스 / 스토리 / 크레딧 | 각 화면의 기존 "메인 메뉴로" 핸들러 호출 |

일시정지 메뉴는 코드로 구성 (`GameRoot._build_pause_menu()`) — 라벨/버튼 텍스트는 모두 Localization 키로 등록되어 한·영 양쪽 모두 즉시 지원.

### 💾 이어하기 (`RunPersist` autoload · `user://run_save.json`)

전투 중 어느 순간에 나가도 다음에 메인 메뉴에서 정확히 이어서 시작합니다.

**저장 트리거**:
- 일시정지 메뉴를 열면 스냅샷 캡처, "메인 메뉴로" 누르면 disk 에 commit
- 앱 백그라운드 진입(`NOTIFICATION_APPLICATION_PAUSED` — 홈 버튼/전화 알림)에도 자동 캡처 + commit

**저장 내용**: Player 좌표/HP/XP/레벨/킬·골드, WeaponManager 무기 리스트(이름·레벨·데미지·CD)와 패시브 dict, WaveManager 경과 시간, GameRoot 런 플래그 (lv5 무피해/진화/보스 처치/광고 재부활·골드2배 사용 여부)

**복원 흐름**:
- 메인 메뉴 상단에 "▶ 이어하기 (스테이지 · Lv.N · M:SS)" CTA 자동 표시
- 누르면 `GameData` 의 스테이지/캐릭터/난이도를 저장된 값으로 핀 후 `GameRoot.tscn` 진입
- `_apply_resume()` 이 Player/WeaponManager/WaveManager 상태를 일괄 복원

**저장 삭제** (오래된 dead-state 복원 방지):
- 새 게임 시작 ("PLAY") / 사망 / 승리 / 결과 패널에서 메인 메뉴·재시도 누름

적·투사체·픽업은 저장하지 않음 — WaveManager 가 `_elapsed` 만으로 자연스럽게 적을 다시 spawn 합니다.

### ☁️ PGS Saved Games 클라우드 백업 (`CloudSave` autoload)

기기 변경/재설치 시 메타 진행이 사라지지 않도록 PGS Snapshots 에 자동 백업:

- `GameData.save_data()` 호출마다 cloud-dirty 플래그
- 10초 throttle 로 PGS Snapshots 에 일괄 업로드 (`SNAPSHOT_NAME = nightseed_meta`)
- `NOTIFICATION_APPLICATION_PAUSED` 시점에 `flush()` 로 즉시 동기화
- MainMenu 진입 시 1회 `request_load()` → 클라우드 페이로드를 **safe merge** (gold = max, unlocks = union) 으로 로컬에 합침. 로컬 진행은 절대 파괴하지 않음
- 비-Android / 로그아웃 상태에서는 모든 호출 no-op — 로컬 `user://save_data.json` 만 동작

> 📌 **Play Console**: 이번 빌드 업로드 후 Play Console "Saved games" 활성화 확인 필요. 비활성이면 클라우드 부분이 자동 비활성화되고 로컬 저장만 동작합니다.
