## 📐 v0.21.0 — 모바일 레이아웃 수정

### 🎯 핵심 변경

폰에서 v0.20.0을 돌려본 결과, **세 화면에서 콘텐츠가 화면 일부에만 몰리고 나머지가
검은 빈 공간으로 남는** 문제가 명확하게 보였습니다. 1080×2340 같은 키 큰 폰일수록
심해지는 컨테이너 레이아웃 결함. v0.21.0은 게임플레이는 손대지 않고 이 세 곳만 잡습니다.

---

### 📱 메인 메뉴 — 액션 블록을 화면 중앙으로

이전: 시작 + 6 버튼이 화면 상단 절반에 몰리고 하단 ~1100px가 통째로 검은 공간.

이제: VBox 안의 기존 6px Spacer를 `size_flags_vertical = 3` + `stretch_ratio = 2.0`로
바꾸고 SecondaryRow 뒤에 `stretch_ratio = 1.0` BottomSpacer를 추가했습니다. 결과:

- 타이틀 + 골드 카드는 상단에 그대로
- 시작 / 캐릭터·스테이지·난이도 / 상점·용어집·순위표 블록이 **화면 2:1 비율로 중앙
  아래쪽** (자연스러운 엄지 위치)에 자리
- 화면 어느 영역도 빈 검은 공간이 아님

---

### 🃏 레벨업 카드 — 카드 높이 자연 크기로

이전: 카드 3개가 `size_flags_vertical = 3` 으로 화면을 채워 늘어났는데 내부
콘텐츠(아이콘+제목+스탯+설명+태그+버튼 ≈ 290px) 아래로 **~440px 빈 어두운 공간**.

이제:

- 카드 `size_flags_vertical = 3` 제거, `custom_minimum_size`만 296 → 320px로 살짝 확장
- VBox에 `alignment = 1` (CENTER) — 세 카드가 자연 높이로 쌓이고 화면 중앙에 클러스터링
- 카드 안쪽 빈 공간 사라짐, 카드 사이의 18px 갭만 남음

### ❤ 인게임 HUD — HP/XP를 한 정보 단위로

이전: HP 바와 XP 바 사이 4px 간격에 XP 바가 너무 얇아서(12px) 연관 없어 보임.

이제:
- HP/XP 간격 4 → 2px
- XP 바 높이 12 → 16px
- XP 바에 파란(`Color(0.45, 0.72, 1.0)`) fill 스타일박스 + 어두운 배경 스타일박스
  ([`HUD.gd._init_xp_bar_style`](godot/scripts/ui/HUD.gd))
- HP(동적 색상) ↔ XP(파란) 색상 분리로 한눈에 구분되면서 하나의 묶음으로 보임
